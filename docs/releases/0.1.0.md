# Packwright 0.1.0

Packwright is a compiler for portable coding-agent packs. Define one editable source—rules, memory contract, skills, and workspace—and build native targets for Codex, Claude Code, and Cursor.

`0.1.0` is the first stable release. It keeps migration explicit: preview the plan, inspect generated, carried, rewritten, and excluded paths, then confirm before Packwright writes a destination.

## Launch-critical hardening

- All mechanism references and generated destinations are contained inside their declared roots. Traversal, absolute paths, source-root escapes, and destination symlink escapes are rejected.
- `packwright build` refuses to overwrite an existing pack unless `--force` is explicit.
- `packwright install --force` preserves portable user state under `memory/`, `workspace/`, `knowledge/`, and `sources/`.
- `packwright doctor` verifies managed projection hashes from the artifact lock and repairs only reproducible drift. Portable and live state are not treated as generated output.
- Embedded metadata is portable and does not expose the build machine's absolute source path.
- Emotion Engine sidecar files are installed only when explicitly enabled.

## Public workflow

- `packwright init` creates an editable agent source from a starter template.
- `packwright build` compiles native packs for Codex, Claude Code, and Cursor.
- `packwright install` projects a checked pack into a target directory.
- `packwright migrate` separates a zero-write plan from an explicitly confirmed move.
- `packwright doctor` detects and can repair deterministic Packwright-managed drift.
- `packwright score` evaluates the public pack structure and artifact contract.

Installed targets include `.packwright/spec.json`, the canonical source snapshot, artifact lock, and checker receipt. A relocated target can still be migrated, diagnosed, and scored without its original work or pack directory.

## Install

```bash
python -m pip install packwright==0.1.0
packwright --version
```

Then follow the [README quickstart](../../README.md) or use the [paste-ready coding-agent prompt](../USE_WITH_YOUR_AGENT.md).

## Verification

The release gate runs the source test suite, zero-network and public-tree audits, wheel and sdist builds, package metadata checks, clean-environment installs, all three adapter workflows, and a Codex-to-Cursor migration with integrity verification.

## Boundaries

- A score of `100.0` proves the public structural rules pass; it does not guarantee runtime behavior.
- Real interactive smoke tests in all three coding runtimes remain an external release check.
- Current adapters are Codex, Claude Code, and Cursor. New adapters land when they pass the checker.
- Packwright is local tooling, not cloud sync.
- Packwright itself makes no network requests and sends no telemetry. A coding runtime's own data policy still applies.
