# Packwright 0.1.0rc1

Packwright is a compiler for agent packs. Define one agent—rules, memory contract, skills, and workspace—and build native packs for Codex, Claude Code, and Cursor.

Build your agent once. Carry it everywhere. Migration is the proof: Packwright recompiles runtime-native files, carries portable state, and reports what the destination cannot carry before it writes anything.

This release candidate is for external installation, migration, and real-runtime testing before stable `0.1.0`.

## Highlights

- `packwright init` creates an editable agent source from a starter template.
- `packwright build` compiles native packs for Codex, Claude Code, and Cursor.
- `packwright install` projects a checked pack into a target directory.
- `packwright migrate` separates a zero-write plan from an explicitly confirmed move.
- `packwright doctor` detects and can repair deterministic Packwright-managed drift.
- `packwright score` evaluates the public pack structure and artifact contract.

## What a migration receipt shows

| Section | Receipt meaning |
|---|---|
| `generated` | Destination-native entry files, skills or rules, and target metadata |
| `carried` | Portable user files copied and verified by SHA-256 |
| `rewritten` | Packwright-managed routing lines changed for the destination |
| `excluded` | Source-runtime files that do not belong in the destination |

The receipt includes planned and installed scores. JSON output uses the public `packwright-migration/v1` schema.

## Self-contained targets

Installed targets include `.packwright/spec.json`, the canonical source snapshot, artifact lock, and checker receipt. A relocated target can still be migrated, diagnosed, and scored without its original work or pack directory.

## Install

```bash
python -m pip install packwright==0.1.0rc1
packwright --version
```

Then follow the [README quickstart](../../README.md) or use the [paste-ready coding-agent prompt](../USE_WITH_YOUR_AGENT.md).

## Known limits

- A score of `100.0` proves the public structural rules pass; it does not guarantee runtime behavior.
- Real interactive smoke tests in all three coding runtimes remain an external release gate.
- Current adapters are Codex, Claude Code, and Cursor. New adapters land when they pass the checker.
- Packwright is local tooling, not cloud sync.
- Packwright itself makes no network requests and sends no telemetry. A coding runtime's own data policy still applies.
