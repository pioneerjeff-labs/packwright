一次构建你的 agent。
带它去任何地方。

Packwright 把一份 agent 定义——规则、记忆、skills 与工作区——编译成 Codex、Claude Code 和 Cursor 的原生 pack。输出是你可以直接阅读的普通文件。

本地优先，零遥测。Packwright 自身不发起网络请求；你的 coding runtime 的数据政策仍然适用。

为什么不只写提示词？

每个 AI coding 工具都要求自己的 instruction 格式。Packwright 换一种方式：把同一份 agent 定义编译成各 runtime 的原生 pack。

可移植

一份 agent 定义，为每个 runtime 投影出一个原生 pack。

可检查

score 为每个 pack 评分，规则就在仓库里。

可修复

doctor --fix 修复确定性的漂移，不碰用户记忆。

熟悉终端，就手动运行；不熟悉，就把一段现成提示词交给 coding agent，直接对话。

把一段提示词粘贴给 Claude Code、Codex 或 Cursor：USE_WITH_YOUR_AGENT.md

手动运行
packwright build # 编译 pack packwright migrate # 携带状态迁移 packwright doctor # 修复漂移

六个公开命令，普通文件，不需要 daemon。快速开始 ↓

快速开始

安装当前 RC，再用三个命令得到一个已安装的 Claude Code target。

python -m pip install packwright==0.1.0rc1
packwright init --template creator -o work/mira
packwright build work/mira --adapter claude-code -o pack/mira-claude
packwright install pack/mira-claude --adapter claude-code --target project/mira-claude

迁移一个正在工作的 agent，也带走它的工作状态。

migrate --dry-run 会报告目标 runtime 无法携带的内容，全程不写入文件。

预览packwright migrate project/mira-claude --to codex --target project/mira-codex --dry-run

查看完整的携带与排除报告，不写入任何内容。

迁移packwright migrate project/mira-claude --to codex --target project/mira-codex --yes

memory/、workspace/、knowledge/ 和 sources/ 会被携带并逐一验证哈希。

验证packwright doctor project/mira-codex
packwright score project/mira-codex

检查已安装的目标并评分，同时留下收据。
