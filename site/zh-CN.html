一次构建 Agent。
随心迁移，无缝运行。

Packwright 能将单一 Agent 定义（包含规则、记忆、Skills 与工作区）直接编译为 Codex、Claude Code 和 Cursor 的原生 Pack。输出结果皆为清晰可读的普通文件。

本地优先，零遥测。Packwright 自身不发起任何网络请求，你的 Coding Runtime 数据策略依然生效。

快速开始 GitHub

为何不只用 Prompt？

每款 AI 编程工具都有各自的 Instruction 格式要求。Packwright 另辟蹊径：将同一份 Agent 定义直接编译为适配各 Runtime 的原生 Pack。

可移植

将单一 Agent 定义映射为各 Runtime 的原生 Pack。

可校验

内置 score 命令为每个 Pack 评分，校验规则与代码同源。

可修复

通过 doctor --fix 修复状态漂移，全程不触碰用户记忆。

自己掌舵，或让 Agent 代驾

爱敲命令，就亲自上手；不想碰终端，就把 Prompt 交给 Coding Agent。聊几句，它替你完成。

迁移完成——project/mira-codex · 安装后评分 100.0。

将以下 Prompt 粘贴至 Claude Code、Codex 或 Cursor 即可：USE_WITH_YOUR_AGENT.md

亲自掌舵
packwright build # 编译 Pack packwright migrate # 携带状态迁移 packwright doctor # 修复状态漂移

仅 6 个公开命令，全普通文件操作，无需后台守护进程（Daemon）。快速开始 ↓

演示路径 · Claude → Codex

迁移 Agent，工作状态一起带走。

migrate --dry-run 会提前报告目标 Runtime 无法兼容的内容——全程仅预览，不写入任何文件。

预览packwright migrate project/mira-claude --to codex --target project/mira-codex --dry-run

查看完整的携带与排除报告，不执行任何写入。

迁移packwright migrate project/mira-claude --to codex --target project/mira-codex --yes

携带 memory/、workspace/、knowledge/ 与 sources/，并进行逐一哈希验证。

验证packwright doctor project/mira-codex
packwright score project/mira-codex

检查目标安装目录并进行评分，自动生成可核验的迁移记录。

选择 Runtime，快速开始。

安装当前 RC 版本，只需三条命令即可构建并安装原生 Target。默认选择 Claude。

python -m pip install packwright==0.1.0rc1
packwright init --template creator -o work/mira
packwright build work/mira --adapter claude-code -o pack/mira-claude
packwright install pack/mira-claude --adapter claude-code --target project/mira-claude
