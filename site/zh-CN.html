一次构建 agent，
随处皆可运行。

Packwright 能将单一 agent 定义（包含规则、记忆、skills 与工作区）直接编译为 Pi、Codex、Claude Code 和 Cursor 的原生 pack。迁移时，记忆、工作区、知识与来源索引也会随 agent 一并带走。

本地优先，零遥测。Packwright 自身不发起任何网络请求，你的 coding runtime 数据策略依然生效。

从你的需求出发

三种预设 agent 模板，满足你多样的定制化需求。

Packwright 提供三种 preset，分别覆盖常见高频需求。支持自定义他们的职责，能力、语气、边界、情绪反馈，回答几个简单问题，就能快速完成角色创建。

代码

天才工程师：擅长编写、审查、调试、测试并交付技术工作。

工作

全能助手：规划项目、起草产出、理清决策并推动后续执行。

陪伴

私人秘书：支持日常安排、生活决策、出行计划与情感陪伴。

需求更特别？可以直接描述自己的 agent，也可以用 packwright adopt 盘点已有 agent 进行迁移适配。

为何不只用 prompt？

每款 AI 编程工具都有各自的 instruction 格式要求。Packwright 另辟蹊径：将同一份 agent 定义直接编译为适配各 runtime 的原生 pack。

可移植

将单一 agent 定义映射为各 runtime 的原生 pack。

可校验

score 检查结构，doctor 另外报告 runtime 就绪度。

可修复

通过 doctor --fix 修复状态漂移，全程不触碰用户记忆。

自己掌舵，或让 agent 代驾

爱敲命令，就亲自上手；不想碰终端，就把 prompt 交给 coding agent。聊几句，它替你完成。

亲自掌舵
packwright draft-character # 描述自己的 agent packwright adopt # 盘点已有 agent packwright build # 编译原生 pack packwright migrate # 携带状态迁移

仅 8 个公开命令，全普通文件操作，无需后台守护进程（daemon）。快速开始 ↓

演示路径 · Claude Code → Codex

迁移 agent，工作状态一起带走。

migrate --dry-run 会提前报告目标 runtime 无法兼容的内容——全程仅预览，不写入任何文件。

预览packwright migrate project/nova-claude --to codex --target project/nova-codex --dry-run

查看完整的携带与排除报告，不执行任何写入。

迁移packwright migrate project/nova-claude --to codex --target project/nova-codex --yes

携带 memory/、workspace/、knowledge/ 与 sources/，并进行逐一哈希验证。

验证packwright doctor project/nova-codex
packwright score project/nova-codex

分别报告结构检查与 runtime 就绪证据，并生成可核验的迁移记录。

选择 runtime，快速开始。

安装当前版本，选择 preset 和名字，再构建一个原生 target。

python -m pip install packwright==0.3.0
packwright init --template code --name Nova -o work/nova
packwright build work/nova --adapter pi -o pack/nova-pi
packwright install pack/nova-pi --adapter pi --target project/nova-pi
